Version 0.26.0 (19.04.2016)

Bugfixes

  • Simulation
    • Fixed crash when rerouting a large number of vehicles in parallel. Issue #2169,Issue #2180
    • Fixes related to zipper nodes.
      • Fixed deadlock Issue #2075
      • Fixed collision Issue #2165
      • Fixed undesired non-determinism.
    • Fixed collision of a vehicle with itself when departing at an edge that forms a tight circle. Issue #2087
    • Fixed bug that was causing wrong vehicle counts in summary output and was preventing the simulation from terminating automatically when continuing from a loaded state. Issue #1494
    • Fixed route errors and crashing when continuing from a loaded state and using routing devices. Issue #2102
    • Fixed bug that was causing false positives when calling traci.inductionloop.getLastStepVehicleIds and traci.inductionloop.getLastStepVehicleNumber. Issue #2104
    • Induction loop detectors now count vehicles which occupy the detector position during insertion. Issue #1468
    • Fixed collisions when using using continuous lane-changing.
    • Fixed bug that was causing erratic emission behavior for stopped vehicles when using the PHEMlight emission model. Issue #2109
    • Fixed unsafe traffic light plans when building networks without exclusive left-green phase. Issue #2113
    • When using the striping model, pedestrians now avoid moving with near-zero speed. Issue #2143
    • Fixed pedestrian collisions. Issue #2145
    • Intersections with more than 64 connections can now be loaded.
    • When approaching a double-connection vehicles now prefer the lane with the prioritized connection. Issue #2160
    • Fixed collision at double-connection. Issue #2170
    • The default vehicle class is now passenger when using the default vehicle type DEFAULT_VEHTYPE. Issue #2181
    • Fixed stuck vehicles when teleporting past an closed edge with a rerouter. Issue #2194
    • Fixed invalid route lengths in tripinfo-output. Issue #2201
    • Rerouters rerouters that combine closingReroute and destProbReroute now only apply to vehicles that are affected by the closed edges. Issue #2208
    • Rerouting devices can now be specified with generic parameters. Issue #2209
    • Fixed invalid waiting position of pedestrians after walking. Issue #2220
    • Fixed detector data for teleporting vehicles. Issue #1452
    • Fixed meandata for circular networks. Issue #1032
  • SUMO-GUI
    • Fixed bug that was causing slightly exaggerated exit times to be shown when activating Show Link Items from the vehicle context menu. Issue #2022
    • Fixed flickering brake lights due to small random decelerations. Issue #2177
    • Areal detectors can now be hidden by setting their size exaggeration to 0.
    • Fixed invalid occupancy value for lane area detectors (was exaggerated by a factor of 100). Issue #2217
    • Fixed crashing related to showing and tracking parameters of arrived vehicles. Issue #2226,Issue #2227
    • Fixed glitch when drawing rail carriages on edges with customized length. Issue #2192
    • Fixed coordinate update without mouse movement. Issue #2076
    • Fixed time display switch in initial view. Issue #2069
    • Vehicle shape and size are now correctly updated when set via TraCI. Issue #2791
  • MESO
    • Fixed bug that broke --meso-multi-queue behavior (regression in 0.25.0). Issue #2150
    • vClass-specific speed limits are now used. Issue #2205
    • tripinfo-output now contains valid timeLoss values. Issue #2204
    • Fixed invalid travel time computation during simulation routing (was averaging segments instead of vehicles).
  • MESO-GUI
    • Fixed crash. Issue #2187
    • Coloring vehicles by selection is now working. Issue #2149
  • NETEDIT
    • Fixed bug that made it impossible to modify numerical attributes (lane numbers, phase duration etc.) on some computers. Issue #1966
    • Fixed error when modifying signal plans for joined traffic lights. Issue #2185
    • Fixed invalid edge length attribute when inspecting networks without internal links. Issue #2210
    • Fixed bug where junctions with uncommon shapes could not be selected.
  • NETCONVERT
    • Fixed bug that was causing unsafe internal junctions to be built. Issue #2086, Issue #2097
    • Fixed bug that was causing z-information to become corrupted. Thanks to Mirco Sturari for the patch.
    • Fixed bug where pedestrians never got the green light when loading a .net.xml file and adding pedestrian crossings.
    • Fixed bug where pedestrian walkingarea edges were missing. (This could cause invalid routes to be generated). Issue #2060
    • Multiple connections from the same edge to the same target lane can now be set in post-processing (i.e. after removal of geometry-like nodes). Issue #2066
    • Option --tls.guess-signals now respects option --tls.unset. Issue #2093
    • Fixed invalid traffic light plans for networks with pedestrian crossings. Issue #2095
    • Loading custom traffic light plans now correctly affects the building of internal junctions. Issue #2098
    • Several fixes in regard to OpenDrive networks:
      • Added missing contactPoint attribute when writing OpenDrive networks.
      • Fixed geometry of lanes within intersections when writing OpenDrive networks.
      • Fixed geometry of lanes when importing imprecise OpenDrive networks Issue #2085.
    • Option --street-sign-output now writes xsd-conforming output.
    • Fixed bugs that were causing invalid TLS plans to be generated for joined traffic lights. Issue #2106, Issue #2107
    • Fixed crash when importing OSM networks related to self-looping edges. Issue #2135
    • Fixed bug that was causing invalid junction shapes and extremely large network boundaries. Issue #2141
    • Fixed crashing (on Windows) and invalid traffic lights (Linux) when loading a .net.xml file and adding splits. Issue #2147
    • Fixed invalid connections at edges with vClass-exclusive lanes. Issue #2157, Issue #2158
    • Fixed invalid traffic light plans for node type traffic_light_right_on_red. Issue #2162
    • Fixed unsafe junction logic when using custom tls plans with node type traffic_light_right_on_red. Issue #2163
    • Connections from lanes that are added during ramp guessing can now be specified. Issue #2175
    • User-defined connections are no longer discarded at guessed ramps. Issue #2196
    • Fixed error when guessing overlapping off-ramps. Issue #2213
    • Fixed error when computing edge shapes with unusual input geometries. Issue #2218
  • TraCI
    • Multiple fixes to the C++ TraCI library
      • commands gui.setScheme, gui.getScheme, inductionloop.getVehicleData
      • various set-commands were not consuming all result bytes and thus corrupted the message stream. Thanks to Alexander Weidinger for the patch. Issue #2007
    • Fixes to TraaS functions Simulation_getDistance2D and Simulation_getDistanceRoad. Issue #2127
    • Fixed crash when using vehicle command move to VTD. Issue #2129
    • vehicle command move to VTD can now position vehicles on internal lanes. Issue #2130
    • Commands that return the road distance no longer return an exaggerated value in networks without internal links (distances across intersections were counted twice).
    • vehicle command distance now takes the depart position into account (was using 0 before). Issue #2203
  • Tools
    • Fixed error that prevented Tools/Visualization#plot_net_dump.py from running.
    • Fixed import errors in vehrouteDiff.py (formerly undocumented tool).
    • netdiff.py now correctly handles non-ascii characters (i.e. international street names). Issue #2112
    • netdiff.py now handles pedestrian crossings. Issue #1893

Enhancements

  • Simulation
    • MESO and MESO#MESO-GUI are now open.
    • Can now simulated rail road crossings (see NETCONVERT enhancement below). Vehicles will get a red light whenever a train is approaching. Issue #1744
    • Added option --emission-output.precision <INT> for configuring the numerical precision of vehicle emissions. Issue #2108
    • Added new departPos value last which can be used to maximize flow and still maintain vehicle ordering. Issue #2025
    • Rerouters now apply to vehicles that are already on the rerouter edge at the start of the active interval. Issue #2207
    • Vehicles that are equipped with a rerouting device now incorporate knowledge about the current traffic state when computing a new route due to encountering a rerouter object. Issue #2197
    • Parking vehicles are no included in the FCD-Output.
    • Riding persons and containers are now included in FCD-Output and Netstate-Output. Issue #1726, Issue #1727
    • The car following model can now be defined using the new vType-attribute carFollowModel (as alternative to using a child XML-element). Issue #1968
    • Added new PHEMlight version. Issue #2206
  • SUMO-GUI
    • Adjusted zooming distance when centering on simulation objects to make objects easier to find. Issue #2146
    • Added option --waiting-time-memory <TIME> for tracking accumulated waiting time of vehicles. The accumulated waiting time (seconds of waiting time within the configured interval, default 100s) can be inspected in the vehicle's parameter window and the vehicles can be colored according to this value. Issue #999
    • Lanes can now be colored by routing device assumed speed. This is an exponential moving average of mean travel speeds used for dynamic rerouting. Issue #2189
  • MESO
    • Added option --meso-tls-penalty <FLOAT> as an alternative way to model the delay effects of traffic lights. When this option is set to a positive value, the expected delay time for each controlled link (based on red duration and cycle duration) is added to the travel time, multiplied with the <FLOAT> argument. By calibrating the parameter, the quality of TLS coordination can be modeled. Issue #2199
  • MESO-GUI
    • Coloring vehicles by depart delay is now working. Added depart delay to the vehicle parameter dialog.
    • Added event time, entry time and block time to the vehicle parameter dialog. These values record when a vehicle leaves, entered and was blocked on an edge segment.
  • NETCONVERT
    • Added new node type rail_crossing to model behavior at a rail road crossings. Issue #1744
  • POLYCONVERT
    • Added default typemaps similar to netconvert. Issue #1853
  • DUAROUTER
    • Added person trips and IntermodalRouting
    • When a flow has a stop definition with attribute until, the time is shifted for each successive vehicle in the flow. Issue #1514
  • MAROUTER
    • Added bulk routing and better OD cell handling for speed improvements. Issue #2167
  • TraCI
    • The python client is now thread safe when using multiple connections in parallel. Each opened connection returns an independent TraCI instance. Issue #2091
    • Added support for vehicle commands to the C++ TraCI library Thanks to Alexander Weidinger for the patch. Issue #2008
    • Added new TraaS commands Edge.getLastStepPersonIDs, Person.getNextEdge, Vehicle.getRouteIndex, Vehicle.getStopState, Vehicle.isStopped and some more stop-related vehicle commands.
    • The angle argument of vehicle command move to VTD now overrides the vehicle angle for drawing and fcd-output. Issue #2131
    • Added new simulation command save state 0x95 for saving the current simulation state. Issue #2191
  • Tools
    • tls_csv2SUMO.py now handles controlled edges within a joined traffic light definition automatically.
    • Added option --ignore-connections to netcheck.py. This can be used to compute all components in the node graph without considering lane-to-lane connections.
    • Added option --symmetrical to generateBidiDistricts.py (previously undocumented tool). This can improve opposite-finding in conjunction with option --radius.
    • Added option --geo to route2poly.py. This can be used to visualize routes from one network within another network.

Other

  • Miscellaneous
    • Visual Studio project files have been updated MSVC12. While it is still possible to build SUMO with MSVC10, this support may be dropped in the future in favor of C++11.
  • Simulation
    • When saving simulation state as XML, lane elements now contain their id for easier inspection.
    • The departPos values maxSpeedGap, pwagSimple and pwagGeneric were removed since they never worked as intended.
    • Option lanechange.allow-swap is now deprecated.
  • SUMO-GUI
    • Vehicle type parameters were moved to a separate dialog box (Show Type Parameter). Issue #2133
  • TraCI
    • TraCI version is now 11.
  • Documentation
    • Added page on saving and loading simulation state
    • Arrival parameters are now documented
    • Extended NETEDIT documentation.
    • Described netdiff.py, a tool for comparing networks which was undocumented for a long time.
    • Added page on modelling networks for motorway simulation. In particular on-off-ramps
    • Added new overview page for usage of elevation data
    • Added documentation on influencing the simulation via SUMO-GUI
    • Added detailed License information.
    • All applications report some build configuration when called without options. Issue #2118
